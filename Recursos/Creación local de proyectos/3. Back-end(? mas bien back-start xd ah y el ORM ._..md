Para que nuestro proyecto de Next.js pueda comunicarse con la base de datos PostgreSQL de forma sencilla y aprovechando al máximo TypeScript, usaremos Prisma. Prisma es un ORM (Object-Relational Mapper) que nos permite leer y escribir en la base de datos usando código, en lugar de escribir sentencias SQL puras.

En la primera pestaña de tu terminal (donde creamos el proyecto de Next.js), detén el servidor de desarrollo por un momento (presionando `Ctrl + C`) y ejecuta estos comandos para instalar las herramientas de Prisma:
```Bash
npm install prisma --save-dev
npm install @prisma/client
```
Una vez que termine la instalación, inicializa Prisma ejecutando:
```Bash
npx prisma init
```
Este último comando acaba de hacer dos cosas muy importantes en tu proyecto:

1. Creó una carpeta llamada `prisma` con un archivo `schema.prisma` adentro (aquí definiremos la estructura de nuestras tablas).
    
2. Creó un archivo oculto llamado `.env` en la raíz de tu proyecto (aquí guardaremos información sensible, como las contraseñas).
    

Abre ese nuevo archivo `.env` en tu editor de código. Verás que Prisma ha puesto una variable de entorno de ejemplo llamada `DATABASE_URL`.

Basado en las credenciales exactas que configuramos hace un momento en nuestro archivo `docker-compose.yml`, ¿cómo crees que debería quedar escrita esa URL de conexión en tu archivo `.env`?

(Pista: el formato que espera PostgreSQL es `postgresql://USUARIO:PASSWORD@localhost:5432/BASEDEDATOS?schema=public`)

# Primer modelito para nuestra BD
luego de hacer la conexión a la instancia de la bd podremos crear nuestra primera tabla para probar que la conexión funciona y empezar a darle forma a nuestro backend. Abre el archivo `prisma/schema.prisma`. Notarás que ya tiene configurado el cliente y el proveedor (que por defecto debería decir `postgresql`).

Vamos a añadir un modelo básico al final de ese archivo. Por ejemplo, una tabla para guardar usuarios:
```Typescript
model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String?
  createdAt DateTime @default(now())
}
```

Guarda el archivo. En este momento, Prisma sabe cómo queremos que sea nuestra base de datos, pero nuestra base de datos en Docker todavía está vacía. Necesitamos sincronizarlas.

Para hacer esto, utilizamos las "migraciones", que son como un historial de versiones para tu base de datos.

¿Conoces o intuyes cuál es el comando de Prisma que usamos en la terminal para crear y aplicar nuestra primera migración en el entorno de desarrollo? _(Pista: empieza con `npx prisma migrate...`)_

>[!INFO] La respuesta es...
>npx prisma migrate dev

Cuando ejecutes `npx prisma migrate dev` en tu terminal, Prisma te pedirá que le des un nombre a esta migración. Puedes ponerle algo descriptivo como "init" o "crear_tabla_usuarios".

Este comando hace la magia: lee tu archivo `schema.prisma`, se conecta a tu PostgreSQL en Docker y crea la tabla real. Además, genera el "Prisma Client", que son los tipos de TypeScript que te permitirán autocompletar el código cuando quieras guardar o buscar usuarios desde Next.js.

Con esto, nuestra base de datos y nuestro backend están listos para trabajar juntos.
