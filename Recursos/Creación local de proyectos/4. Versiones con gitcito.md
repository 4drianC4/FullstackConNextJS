Una ventaja de haber usado `create-next-app` en la Fase 1 es que, por defecto, ya inicializó un repositorio local de Git por nosotros y creó un archivo `.gitignore` (que evita que subamos cosas pesadas como la carpeta `node_modules` o archivos sensibles como el `.env`).

Para subir esto a GitHub, primero necesitamos preparar nuestros cambios locales y guardarlos en el historial (hacer un _commit_).

¿Recuerdas cuáles son los dos comandos básicos de `git` que usamos en la terminal para añadir todos los archivos modificados al área de preparación (staging) y luego guardar ese _commit_ con un mensaje?

Ahora, para subir esto a GitHub y terminar nuestro proceso, sigue estos 3 pasos finales:

1. **Crea el repositorio:** Ve a tu cuenta de GitHub y crea un **Nuevo Repositorio**. Dale el nombre que quieras, pero es crucial que lo dejes **completamente vacío** (no marques las casillas para agregar README, .gitignore o licencia, ya que nosotros ya tenemos los nuestros localmente).
    
2. **Copia la URL:** GitHub te mostrará una pantalla con instrucciones. Copia la URL de tu nuevo repositorio (se verá algo como `https://github.com/tu-usuario/mi-proyecto.git`).
    
3. **Conecta y sube:** En tu terminal, ejecuta estos dos comandos para conectar tu código local con el repositorio remoto y enviar los archivos:

```Bash
git remote add origin URL_DE_TU_REPOSITORIO
git push -u origin main
```
